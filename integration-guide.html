STEP 01 — GCP

1. GCP 프로젝트 설정

1

새 프로젝트 생성

GCP 콘솔 상단 프로젝트 선택 → 새 프로젝트 → 이름 입력 (예: campaign-dashboard).

2

API 활성화

"API 및 서비스" → "라이브러리" 에서 다음 두 API 검색 후 활성화:

  • Google Sheets API
  • Google Drive API
3

자격증명 생성

"API 및 서비스" → "사용자 인증 정보" → "+ 사용자 인증 정보 만들기":

  • API 키 — 공개 시트라면 이것만으로 충분. 클라이언트에서 직접 호출.
  • OAuth 2.0 클라이언트 ID — 사용자별 시트에 접근할 때.
  • 서비스 계정 — 백엔드/Apps Script에서 비대화식 접근.
주의: API 키는 도메인 제한(HTTP referer)을 반드시 걸어두세요. 노출되면 누구나 사용 가능합니다.
STEP 02 — SHEETS

2. Google Sheets API 연동

가장 단순한 방식은 시트를 "링크가 있는 모든 사용자 보기"로 공유하고 API 키로 직접 읽는 것입니다.

2-1. 시트 공유 설정

  • 스프레드시트 우측 상단 공유 → "링크가 있는 모든 사용자" → "뷰어"
  • 또는 서비스 계정 이메일(xxx@xxx.iam.gserviceaccount.com)을 뷰어로 추가

2-2. 시트 ID 확인

시트 URL에서 /d/ 다음에 오는 긴 문자열입니다.

// 예시 URL
https://docs.google.com/spreadsheets/d/1aBcDeFgHiJkLmNoPqRsTuVwXyZ0123456789/edit
                                       └────────── SHEET_ID ──────────┘

2-3. JS로 데이터 읽기

const SHEET_ID = "1aBcDeFg...";
const RANGE    = "campaign-perf!A1:K";
const API_KEY  = "AIzaSy...";  // referer 제한 권장

async function fetchRows() {
  const url = `https://sheets.googleapis.com/v4/spreadsheets/${SHEET_ID}/values/${RANGE}?key=${API_KEY}`;
  const res = await fetch(url);
  const { values } = await res.json();
  const [headers, ...data] = values;
  return data.map(row =>
    headers.reduce((obj, key, i) => ({ ...obj, [key]: row[i] }), {})
  );
}
대량 데이터: 5,000 행 이상이면 batchGet 또는 페이지네이션을 고려하세요. Apps Script 사용 시 SpreadsheetApp.getRange(...).getValues() 로 가장 빠릅니다.
STEP 03 — DRIVE

3. Drive 이미지 연동

광고 소재 이미지는 Google Drive의 특정 폴더에 정리되어 있고, 파일명 또는 메타데이터로 시트의 캠페인/광고그룹과 매핑됩니다.

3-1. 폴더 구조 권장

/creatives
├── 2026-Q2/
│   ├── [BR_MAIN][NVR-SA][PC]_v1.png
│   ├── [PR_NEW][META][MO]_v3.jpg
│   └── [EV_MAY][YT][MO]_v1.png
└── 2026-Q1/

파일명을 [캠페인키][매체키][기기]_버전.확장자 패턴으로 통일하면 매핑이 자동화됩니다.

3-2. 폴더 내 파일 목록 가져오기

const FOLDER_ID = "1xYz...";  // 드라이브 폴더 URL 끝부분

async function listCreatives() {
  const q = `'${FOLDER_ID}' in parents and mimeType contains 'image/'`;
  const url = `https://www.googleapis.com/drive/v3/files`
            + `?q=${encodeURIComponent(q)}`
            + `&fields=files(id,name,thumbnailLink,webContentLink)`
            + `&key=${API_KEY}`;
  const res = await fetch(url);
  const { files } = await res.json();
  return files;
}

3-3. 이미지를 <img>로 표시

Drive 이미지는 thumbnailLink(소형) 또는 다음 형식의 직접 URL로 임베드합니다:

// 1) 썸네일 — 빠르고 인증 불필요 (공유 폴더)
<img src="https://drive.google.com/thumbnail?id=FILE_ID&sz=w400"/>

// 2) 직접 임베드 — 풀 사이즈
<img src="https://lh3.googleusercontent.com/d/FILE_ID=w800"/>

// 3) iframe 미리보기 — PDF/문서
<iframe src="https://drive.google.com/file/d/FILE_ID/preview"/>
주의: Drive 이미지를 외부 사이트에 임베드하려면 파일이 "링크가 있는 모든 사용자에게 공개" 상태여야 합니다. 비공개 파일은 OAuth 토큰으로 다운로드 후 Blob URL로 렌더링해야 합니다.
STEP 04 — MAPPING

4. 시트 스키마 매핑

스프레드시트의 컬럼을 대시보드 내부 모델에 매핑합니다. 현재 시트 컬럼 구조 기준:

시트 컬럼내부 키타입비고
A 주차weekstring예: "5월 1주차" — 표시용
B 날짜dateYYYY.MM.DD정렬/필터 키
C 매체mediaenum네이버 S/A, 구글, Meta…
D 캠페인campaignstring그룹핑 차원
E 광고그룹adGroupstring
F 기기devicePC | MO
G 노출수impressionsint측정 지표
H 클릭수clicksint
I 비용(KRW)costint원 단위
J 전환수conversionsint
K 매출revenueint원 단위

파생 지표(CTR, CPC, CPA, ROAS)는 클라이언트에서 계산합니다:

CTR  = clicks / impressions
CPC  = cost / clicks
CPA  = cost / conversions
ROAS = revenue / cost
STEP 05 — AUTH

5. 인증 패턴

패턴 A — Apps Script 웹앱 (가장 간단)

시트의 확장 프로그램 → Apps Script에 코드를 붙여 넣고 웹앱으로 배포. 별도 키 관리 불필요.

// Code.gs
function doGet(e) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet()
                          .getSheetByName("campaign-perf");
  const values = sheet.getDataRange().getValues();
  return ContentService
    .createTextOutput(JSON.stringify(values))
    .setMimeType(ContentService.MimeType.JSON);
}

대시보드에서는:

const rows = await fetch("https://script.google.com/macros/s/XXX/exec")
                  .then(r => r.json());

패턴 B — 서비스 계정 (백엔드 + 비공개 시트)

// Node.js / googleapis
import { google } from 'googleapis';
const auth = new google.auth.GoogleAuth({
  keyFile: './service-account.json',
  scopes: [
    'https://www.googleapis.com/auth/spreadsheets.readonly',
    'https://www.googleapis.com/auth/drive.readonly',
  ],
});
const sheets = google.sheets({ version: 'v4', auth });
const res = await sheets.spreadsheets.values.get({
  spreadsheetId: SHEET_ID,
  range: 'campaign-perf!A:K',
});

패턴 C — OAuth 2.0 (사용자별 시트)

각 사용자가 자신의 시트를 보게 하려면 OAuth 동의 화면을 통해 토큰을 받습니다. Google Identity Services(GIS) JS 라이브러리 사용 권장.

STEP 06 — SYNC

6. 자동 동기화

  • 폴링: 대시보드에서 5~10분마다 fetch — 가장 단순.
  • Pub/Sub 푸시 알림: 시트 변경 시 Drive Changes API가 웹훅 전송. 실시간 가능.
  • Apps Script 트리거: 시트 수정 시 자동으로 캐시 무효화 호출.
// 클라이언트 폴링 예시
setInterval(async () => {
  const fresh = await fetchRows();
  store.setRows(fresh);
}, 5 * 60 * 1000);
